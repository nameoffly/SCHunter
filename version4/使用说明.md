# SVHunter v4 前端界面使用说明

## 1. 环境要求

| 组件 | 要求 |
|------|------|
| Node.js | >= 18 |
| Python | >= 3.11 |
| pip | 最新版本 |
| 浏览器 | Chrome / Edge / Firefox 等现代浏览器 |

> 如需实际运行 SVHunter 管线（而非仅查看演示），还需要提前配置好 SVHunter 的 conda 环境（TensorFlow 2.12.1、pysam 等），详见 `SVHunter-1/README.md`。

---

## 2. 安装依赖

```bash
# 进入 version4 目录
cd version4

# 安装前端依赖
npm install

# 安装后端依赖
pip install -r server/requirements.txt
```

---

## 3. 启动服务

### 3.1 仅启动前端（演示模式）

不启动后端时，前端仍可正常访问，Results 页面会自动使用内置的 Mock 演示数据。

```bash
cd version4
npm run dev
```

打开浏览器访问 **http://localhost:5173**。

### 3.2 前端 + 后端（完整模式）

需要两个终端窗口分别启动：

**终端 1 — 启动后端**：

```bash
cd version4
uvicorn server.main:app --reload --port 8000
```

启动成功后，可通过 http://localhost:8000/api/health 验证，应返回 `{"status":"ok"}`。

**终端 2 — 启动前端**：

```bash
cd version4
npm run dev
```

前端开发服务器会自动将 `/api` 请求代理到后端 `localhost:8000`，无需额外配置。

### 3.3 指定 VCF 结果目录

如果已有 SVHunter 跑出的 VCF 结果文件，可以通过环境变量告诉后端去哪里查找：

```bash
# 用冒号分隔多个目录
VCF_SEARCH_DIRS=/path/to/vcf_output1:/path/to/vcf_output2 \
  uvicorn server.main:app --reload --port 8000
```

也可以在 Results 页面右上角的输入框中直接填写 VCF 目录路径。

## 4. 三个页面功能说明

### 4.1 Pipeline 运行控制台 (`/pipeline`)

用于通过网页界面运行 SVHunter 的两个命令行操作。

**Generate 模式（特征提取）**

对应命令：`python SVHunter.py generate <bam> <output> <threads> <chroms>`

| 参数 | 说明 |
|------|------|
| BAM 文件路径 | 长读长比对文件的绝对路径 |
| 输出目录 | 存放生成的 `.npy` 特征数据的目录 |
| 线程数 | 并行处理线程数，默认 4 |
| 染色体列表 | 可选，逗号分隔（如 `1,2,3`），留空则检测全部染色体 |

**Call 模式（SV 检测）**

对应命令：`python SVHunter.py call <model> <data> <bam> <predict> <vcf> <threads> <chroms> <gpus>`

| 参数 | 说明 |
|------|------|
| 模型权重路径 | `.h5` 模型文件路径，默认 `./model_predict.h5` |
| 特征数据目录 | Generate 步骤输出的目录 |
| BAM 文件路径 | 同 Generate 中使用的 BAM 文件 |
| 预测输出目录 | 存放模型预测结果的目录 |
| VCF 输出目录 | 最终 VCF 文件输出目录 |
| 线程数 | 默认 4 |
| GPU 数量 | 默认 1，设为 0 则使用 CPU |
| 染色体列表 | 同上 |

**使用流程**：
1. 选择 Generate 或 Call 模式
2. 填写参数表单
3. 点击运行按钮
4. 右侧面板会实时显示运行日志和状态（通过 SSE 推送）
5. 完成后状态变为 SUCCESS 或 FAILED

> 注意：运行 Pipeline 需要后端已启动，且服务器上已配置好 SVHunter 的 Python 环境。

### 4.2 算法流程可视化 (`/algorithm`)

此页面展示 SVHunter 四阶段检测流水线的详细说明，无需后端。

**顶部流程图**：横向展示 BAM → 特征提取 → 模型预测 → 候选聚类 → VCF 的完整流水线。

**下方详情卡片**（点击展开）：

| 阶段 | 内容 |
|------|------|
| Stage 1 - 特征提取 | 10Mb 分块、CIGAR/SA-tag 解析、20 通道特征张量构建、Z-score 归一化 |
| Stage 2 - 模型预测 | CNN 编码器、CBAM 注意力、7 层 Transformer、sigmoid 评分 |
| Stage 3 - 候选聚类 | 阳性窗口合并、MeanShift 聚类、支持读数阈值计算 |
| Stage 4 - 基因分型与输出 | 贝叶斯+EM+深度比三重投票、VCF v4.2 输出 |

每个阶段展开后包含：详细步骤列表 + 关键参数表。

### 4.3 样本结果展示 (`/results`)

用于浏览已完成检测的样本结果。

**页面组成**：

- **统计摘要卡片**：总 SV 数量、各类型（DEL/INS/DUP/INV/BND）数量、中位长度
- **SV 类型分布饼图**：直观展示各类型占比
- **染色体分布柱状图**：各染色体上检出的 SV 数量
- **VCF 记录表格**：支持按类型筛选和关键词搜索

**数据来源**：
- 后端启动且配置了 VCF 目录时 → 自动加载真实数据
- 后端未启动或无数据时 → 自动 fallback 到内置的 Mock 演示数据
- 页面右上角可直接输入 VCF 目录路径加载指定位置的数据

## 7. 常见问题

**Q: 前端启动后页面空白？**
确认 `npm install` 已执行，且 Node.js 版本 >= 18。

**Q: Pipeline 页面点击运行报错？**
确认后端已启动（`uvicorn server.main:app --reload --port 8000`），且 SVHunter 的 conda 环境已激活。

**Q: Results 页面只显示演示数据？**
需要启动后端并通过 `VCF_SEARCH_DIRS` 环境变量或页面输入框指定包含 VCF 文件的目录。SVHunter 的输出目录结构通常为 `<output>/<sample_name>/<sample_name>_all.vcf`。

**Q: 后端端口冲突？**
修改启动命令中的端口号，同时更新 `vite.config.ts` 中 proxy 的 target 地址。
